# Charts (shadcn Chart + Recharts)

The user said _"add a chart"_, _"show this as a graph"_, _"visualize
these numbers"_, _"build a dashboard"_. Use the shadcn `Chart`
primitive — it's a thin, theme-aware wrapper around **Recharts** that
maps your data series to CSS variables so colors stay consistent with
the rest of the app.

Reference: <https://ui.shadcn.com/docs/components/radix/chart>

## Prerequisites

The chart primitive should already be installed at
`components/ui/chart.tsx`. If not:

```bash
npx shadcn@latest add chart
```

`recharts` is already in `package.json`.

## The two pieces you always need

1. **`ChartConfig`** — a typed map from each data key to a label and
   color. This is what makes the chart theme-aware.
2. **`ChartContainer`** — wraps a Recharts `<ResponsiveContainer>` and
   injects CSS variables (`--color-<key>`) from your config.

Inside the container you compose **Recharts** primitives directly
(`LineChart`, `BarChart`, `Line`, `Bar`, `XAxis`, etc.) plus the shadcn
tooltip/legend helpers (`ChartTooltip`, `ChartTooltipContent`,
`ChartLegend`, `ChartLegendContent`).

## Pattern: bar chart

```tsx
"use client"

import { Bar, BarChart, CartesianGrid, XAxis } from "recharts"
import {
  ChartContainer,
  ChartTooltip,
  ChartTooltipContent,
  type ChartConfig,
} from "@/components/ui/chart"

const data = [
  { month: "Jan", revenue: 4200, refunds: 320 },
  { month: "Feb", revenue: 5100, refunds: 410 },
  { month: "Mar", revenue: 4800, refunds: 290 },
  { month: "Apr", revenue: 6200, refunds: 510 },
]

const chartConfig = {
  revenue: { label: "Revenue", color: "var(--chart-1)" },
  refunds: { label: "Refunds", color: "var(--chart-2)" },
} satisfies ChartConfig

export function RevenueChart() {
  return (
    <ChartContainer config={chartConfig} className="h-64 w-full">
      <BarChart data={data}>
        <CartesianGrid vertical={false} />
        <XAxis dataKey="month" tickLine={false} axisLine={false} />
        <ChartTooltip content={<ChartTooltipContent />} />
        <Bar dataKey="revenue" fill="var(--color-revenue)" radius={4} />
        <Bar dataKey="refunds" fill="var(--color-refunds)" radius={4} />
      </BarChart>
    </ChartContainer>
  )
}
```

Key things to notice:

- `fill="var(--color-revenue)"` — the **`--color-<key>`** variable is
  auto-generated by `ChartContainer` from your `ChartConfig`. You never
  hardcode hex colors in JSX.
- `satisfies ChartConfig` — keeps TS happy without erasing literal keys
  (so `--color-revenue` is type-checked indirectly).
- `"use client"` — charts use refs and ResizeObserver. **Always client.**

## Pattern: line chart

```tsx
import { CartesianGrid, Line, LineChart, XAxis } from "recharts"

export function RevenueLine() {
  return (
    <ChartContainer config={chartConfig} className="h-64 w-full">
      <LineChart data={data} margin={{ left: 12, right: 12 }}>
        <CartesianGrid vertical={false} />
        <XAxis dataKey="month" tickLine={false} axisLine={false} />
        <ChartTooltip content={<ChartTooltipContent />} />
        <Line
          dataKey="revenue"
          type="monotone"
          stroke="var(--color-revenue)"
          strokeWidth={2}
          dot={false}
        />
      </LineChart>
    </ChartContainer>
  )
}
```

## Pattern: area chart with gradient

```tsx
import { Area, AreaChart, CartesianGrid, XAxis } from "recharts"

export function RevenueArea() {
  return (
    <ChartContainer config={chartConfig} className="h-64 w-full">
      <AreaChart data={data}>
        <defs>
          <linearGradient id="fillRevenue" x1="0" y1="0" x2="0" y2="1">
            <stop
              offset="5%"
              stopColor="var(--color-revenue)"
              stopOpacity={0.8}
            />
            <stop
              offset="95%"
              stopColor="var(--color-revenue)"
              stopOpacity={0.1}
            />
          </linearGradient>
        </defs>
        <CartesianGrid vertical={false} />
        <XAxis dataKey="month" tickLine={false} axisLine={false} />
        <ChartTooltip content={<ChartTooltipContent />} />
        <Area
          dataKey="revenue"
          type="natural"
          fill="url(#fillRevenue)"
          stroke="var(--color-revenue)"
        />
      </AreaChart>
    </ChartContainer>
  )
}
```

## Pattern: pie / donut

```tsx
import { Pie, PieChart } from "recharts"

const data = [
  { segment: "free", users: 540, fill: "var(--color-free)" },
  { segment: "pro", users: 210, fill: "var(--color-pro)" },
  { segment: "enterprise", users: 45, fill: "var(--color-enterprise)" },
]

const chartConfig = {
  users: { label: "Users" },
  free: { label: "Free", color: "var(--chart-1)" },
  pro: { label: "Pro", color: "var(--chart-2)" },
  enterprise: { label: "Enterprise", color: "var(--chart-3)" },
} satisfies ChartConfig

export function PlanBreakdown() {
  return (
    <ChartContainer config={chartConfig} className="mx-auto aspect-square h-64">
      <PieChart>
        <ChartTooltip content={<ChartTooltipContent hideLabel />} />
        <Pie data={data} dataKey="users" nameKey="segment" innerRadius={48} />
      </PieChart>
    </ChartContainer>
  )
}
```

For pie/radial charts the per-slice `fill` lives **on the data**
(`fill: "var(--color-free)"`), and the matching key in `chartConfig` is
the slice's `nameKey` value.

## Color palette

`app/globals.css` defines `--chart-1` through `--chart-5` for both
light and dark mode. Always reference colors via those tokens:

```ts
color: "var(--chart-1)" // ✅ theme-aware
color: "#ff6b6b" // ❌ breaks dark mode, drifts from palette
```

If you need more than 5 distinct series, that's usually a sign the
chart is over-loaded — propose a different visualization to the user.

## Tooltip and legend

shadcn ships matched helpers. Use them in pairs inside the chart:

```tsx
import {
  ChartLegend,
  ChartLegendContent,
  ChartTooltip,
  ChartTooltipContent,
} from "@/components/ui/chart"

// inside <BarChart> / <LineChart> / etc:
//   <ChartTooltip content={<ChartTooltipContent indicator="dot" />} />
//   <ChartLegend content={<ChartLegendContent />} />
```

Useful props on `ChartTooltipContent`:

- `indicator`: `"dot" | "line" | "dashed"`
- `hideLabel`, `hideIndicator` — for compact tooltips
- `labelKey`, `nameKey` — override which data field shows where

## Where to put chart components

- Single-use chart for one page: inline in `app/<route>/page.tsx` (as
  long as the file is < 200 lines).
- Reusable / parameterized chart: extract to `components/<name>-chart.tsx`.
- The chart **data fetching** lives in a Server Component or
  Route Handler. The chart **component itself** is `"use client"`. So
  the typical wiring is: server fetches → passes `data` as a prop to
  the client chart component.

## Validating the data

Chart data often comes from an API. Parse it with a zod schema first —
see `.claude/skills/validation/SKILL.md`. A malformed point will
silently render as a gap or NaN; catching it at the boundary is much
cheaper than debugging an empty chart.

```ts
const ChartPointSchema = z.object({
  month: z.string(),
  revenue: z.number(),
  refunds: z.number(),
})
const ChartDataSchema = z.array(ChartPointSchema)
```

## What you must NOT do

- ❌ Import Recharts components without wrapping them in
  `ChartContainer`. You lose theming, tooltips, and dark-mode support.
- ❌ Hardcode hex colors in `fill` / `stroke`. Always go through
  `var(--color-<key>)` from the chart config.
- ❌ Install another charting library (Chart.js, Victory, Nivo,
  visx, ApexCharts). Stop and ask the user first.
- ❌ Forget `"use client"` — the chart will crash on the server.
- ❌ Use `ResponsiveContainer` directly. `ChartContainer` already
  provides it; nesting two breaks layout.

## After adding a chart

```bash
npm run build
```

This catches the most common failure: a server component accidentally
importing a `"use client"` chart in a way Next can't tree-shake.
